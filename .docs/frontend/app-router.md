# Scope

- Covers the current route tree and shared layout files in [`src/app`](../../src/app).
- Focuses on [`src/app/layout.tsx`](../../src/app/layout.tsx) and [`src/app/page.tsx`](../../src/app/page.tsx).

# Current Subdomain Docs

- The repository currently exposes a single root route from [`src/app/page.tsx`](../../src/app/page.tsx); there are no nested route segments yet.
- [`src/app/layout.tsx`](../../src/app/layout.tsx) is the shared root layout and is responsible for importing global CSS, setting `<html lang="en">`, and exporting route metadata.
- Google fonts are initialized in the root layout with `next/font/google` and exposed as CSS variables on the `<html>` element.
- Current metadata is still scaffolded as `"Create Next App"` and `"Generated by create next app"`, so any productization work should update metadata together with homepage content.
- The homepage uses [`next/image`](https://nextjs.org/docs/app/api-reference/components/image) for local assets in `public/`, which is the current pattern for image rendering in the app shell.

# Update Triggers

- Update this file when new routes, nested layouts, or route groups are added under `src/app`.
- Update this file when metadata ownership moves away from the root layout.
- Update this file when the app shell stops being scaffold content and durable route conventions emerge.
